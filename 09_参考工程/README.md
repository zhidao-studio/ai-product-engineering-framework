### 维护者例外授权

此前规则要求高保真批准后再启动实现。维护者于 2026-07-16 明确要求“去修改吧，升级一个小版本”，YouYu 已通过 `DEC-004` 记录受控例外：

- 允许数据库和服务端基础实现先行；
- 不代表高保真已经批准；
- 不允许正式 iOS 页面提前进入实现；
- 不允许扩大到其他业务模块；
- 实现状态必须保持 `implemented_not_validated`。

该例外是人类责任人覆盖阶段检查关卡的可审计证据，而不是删除检查关卡。

- 安全、网络隔离、采集待审和历史敏感信息问题仍未关闭；
- GitHub Actions 账户账单或额度限制仍阻塞远程证据。

## 5. 正式参考工程通过标准

只有下列证据完整，才能把候选资产评估为 `single_project_validated`：

1. 上游产品、体验和工程事实可追溯且人工批准；
2. 项目、阶段和任务 Context 可由新 Agent 独立恢复；
3. 修改边界、参数确认、接口和检查关卡真实生效；
4. 静态、数据库、运行、安全和模拟用户验证完成；
5. iOS 与服务端完成真实端到端路径；
6. 失败、人工修正、成本和遗漏已经记录；
7. 运行结果和 Framework 改进项完成回写；
8. 明确的人类责任人批准最终结论。

当前尚不满足这些条件，正式业务验证必须保持 `not_started`。
