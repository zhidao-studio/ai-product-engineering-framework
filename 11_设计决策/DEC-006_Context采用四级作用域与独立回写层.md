# DEC-006：Context 采用四级作用域与独立回写层

- 状态：已接受
- 日期：2026-07-12
- 影响版本：v0.2 及后续版本

## 背景

v0.1 的 Context 文档同时出现“框架、项目、阶段、任务”和“经验与决策”五个节点，但标题称为四级模型，容易把作用域层级和记忆回写类型混为一谈。

同时，仅仅规定“把上下文外置”仍不足以执行。需要回答：

- 不同信息属于哪个作用域；
- 哪个文件是权威事实源；
- 当前任务应该装配哪些信息；
- 发生冲突、过期和缺失时如何处理；
- 执行经验如何经过审查后进入长期记忆。

## 决定

Context 体系采用“**四级作用域 + 独立回写层**”模型。

### 四级作用域

1. **框架级 Context**：愿景、原则、术语、核心模型和通用规则；
2. **项目级 Context**：项目目标、业务规则、设计、架构、契约、环境和长期状态；
3. **阶段级 Context**：当前生命周期阶段的目标、输入、产物、门禁、依赖和责任人；
4. **任务级 Context**：当前任务的目标、范围、事实引用、允许与禁止事项、风险和验收证据。

### 独立回写层

设计决策、验证证据、失败原因和经验不作为第五个作用域，而是作为跨层记忆资产：

- 决策解释为什么形成当前规则；
- 证据证明当前结论是否成立；
- 经验记录候选改进；
- 经审查后，回写到框架级、项目级、阶段级或任务模板。

## 关键约束

- Context Pack 是从权威事实源装配出的任务快照，不自动成为新的权威事实源；
- 不把整个仓库无差别输入模型，应遵循相关、最小、可追溯和安全原则；
- 自动记忆和执行经验必须先标记为候选，经验证和批准后才能成为强制规则；
- 任务 Context 必须固定引用的路径、版本或提交，避免执行期间事实漂移；
- 冲突不得静默解决，必须形成冲突记录、责任人和处理结果；
- 过期内容必须标记为已替代或归档，不得与当前规则并列有效。

## 影响

后续 Context 工程按以下模块建设：

1. 事实源与状态规范；
2. 项目 Context Pack；
3. 阶段 Context；
4. 任务 Context Pack；
5. 装配、裁剪和安全规则；
6. 冲突、缺失和过期处理；
7. 决策、证据和经验回写；
8. 完整性检查与参考工程验证。

该决定不改变三平面全局模型，只细化 Context Engineering 的内部结构。
