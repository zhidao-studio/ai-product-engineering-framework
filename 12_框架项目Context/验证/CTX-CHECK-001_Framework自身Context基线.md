## 6. P1 问题

### P1-1：尚无自动链接、版本和状态门禁

当前仍以人工确认为主。Harness 里程碑应增加：

- Markdown 内部链接检查；
- 重复目录或同义文档检查；
- Mermaid 基础语法检查；
- Context 元数据完整性检查；
- 版本字段和状态枚举检查；
- 已替代文档不得被新任务引用的检查。

### P1-2：Framework 自应用不能代替业务验证

尚未证明：

- Pack 对产品、设计和代码任务是否足够；
- 填写成本是否适合个人和小团队；
- 敏感数据和企业权限场景是否可用；
- 多平台 Agent 是否能得到一致结果。

### P1-3：部分信息仍有重复维护成本

项目 Context、README、AGENTS 和 Roadmap 都描述当前状态。后续应通过机器元数据或自动生成减少漂移风险。

### P1-4：开源许可证尚未确定

仓库已经提供贡献规范，但未提交 `LICENSE`。许可证必须由项目维护者决定，Agent 不得擅自选择。

> 后续 v0.1.2 已建立专有 LICENSE；本项是历史问题，不再是当前待决事项。
