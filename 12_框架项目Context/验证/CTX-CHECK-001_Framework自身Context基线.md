## 6. P1 问题

### P1-1：尚无自动链接和状态门禁

当前依靠人工读取确认文件存在和口径一致。后续 Harness 阶段应增加：

- Markdown 内部链接检查；
- 重复目录或同义文档检查；
- Mermaid 基础语法检查；
- Context 元数据完整性检查；
- 已替代文档不得被新任务引用的检查。

### P1-2：Framework 自应用不能代替业务验证

本次验证主要证明仓库记忆、导航和一致性管理有效，尚未证明：

- Pack 对产品、设计和代码任务是否足够；
- 填写成本是否适合个人和小团队；
- 敏感数据和企业权限场景是否可用；
- 多平台 Agent 是否能得到一致结果。

### P1-3：部分信息仍有重复维护成本

项目 Context、README、AGENTS 和 Roadmap 都描述当前阶段。当前重复是为了不同受众快速理解，但后续应通过元数据或自动生成减少漂移风险。

### P1-4：提交粒度较细

本轮采用逐文件中文提交，追溯性好，但提交数量较多。后续真实任务建议按照“规范、模板、验证”三个逻辑批次控制提交粒度。
