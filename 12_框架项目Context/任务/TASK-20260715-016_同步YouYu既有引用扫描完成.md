## 1. 目标

同步 YouYu `TASK-009` 的既有引用扫描结果，确认候选账号与个人资料工程规格不能直接套用旧管理员认证和数据模型，并将真实改造边界回写到 Framework。

## 2. 已核验事实

- 旧 `/api/auth/v1` 使用用户名密码登录、注册分离和 `Long` 登录 ID，不满足 App 手机号验证码统一登录注册；
- `sys_user` 混合管理员账号、昵称、头像和手机号，旧 SQL 使用 `BIGINT AUTO_INCREMENT`、字段默认时间和 `ON UPDATE`，不符合 App 用户的独立模型和数据库基础规范；
- `BaseEntity` 及其既有继承对象使用 `Long` 主键和旧审计字段，不能在本业务切片中被静默改造；
- 网关可传递字符串内部身份，但下游 `RequestContext`、`TraceIdFilter` 和 `CurrentUserHelper` 等仍存在 `Long` 类型边界；
- `/api/app/v1/**` 已路由至 App 服务，发送验证码、统一短信登录、协议和地区等公开路径必须在正式实现时显式加入白名单；
- 旧认证 Controller 和依赖会被 App 服务扫描，新增 App 认证入口必须使用 `/api/app/v1/auth/**` 并避免覆盖旧 `/api/auth/v1/**`。

## 3. Framework 结论

本次扫描是工程规格准备的真实输入，不代表数据库迁移、OpenAPI、鉴权改造或业务代码已经完成。后续必须先确认关键业务与安全参数，并建立独立迁移、回滚、触发器、正式 OpenAPI、实现边界和三层验证方案。

## 4. 明确边界

- App 用户账号、个人资料、短信验证码和协议确认与管理员 `sys_user` 分离；
- App 登录身份采用 32 位字符串，不能把 App 数据写入仍以 `BIGINT` 或 `Long` 为前提的旧业务表；
- 不重写或破坏既有管理员认证、AI、文件和系统配置实体；
- 本次只同步扫描事实，不修改 YouYu 业务代码、数据库、网关配置或稳定版本。

## 5. 后续检查关卡

1. 维护者批准高保真并确认验证码、Token、手机号、账号状态、资料、地区与协议版本参数；
2. 正式迁移、回滚和触发器设计通过 DB-CHECK；
3. OpenAPI、错误码、示例数据和客户端字段映射形成唯一权威来源；
4. 字符串身份上下文、网关公开路径、共享会话和下游网络隔离形成可运行验证证据；
5. 受控实现任务才可创建，正式业务实现继续保持 `blocked`。
