## 1. 项目身份

- **仓库**：`zhidao-studio/ai-product-engineering-framework`；
- **一句话定位**：一套开放、跨平台、可验证的 AI 产品工程框架；
- **目标用户**：个人开发者、创业团队、企业研发与创新团队、AI 工程平台维护者；
- **核心问题**：如何让 AI Agent 在人的目标与责任下，跨完整产品生命周期受控、可验证、可持续地参与软件产品生产；
- **当前稳定版本**：v0.1.0；
- **当前建设阶段**：v0.2-A Context 可执行化；
- **默认分支**：`main`；
- **仓库规则入口**：[AGENTS.md](../AGENTS.md)。

## 2. 当前目标与退出标准

### 当前目标

将 v0.1 的 Context 原则转化为可执行的事实源、项目 Pack、阶段 Context、任务 Pack、装配、冲突、回写、检查清单和候选模板。

### v0.2-A 退出标准

一个真实业务任务能够证明：新的人员或 Agent 不依赖历史会话，也能恢复项目背景、识别权威事实、发现冲突、明确范围并完成结果回写。

### 当前验证位置

- Framework 自应用：进行中；
- 业务参考工程：尚未开始；
- 自动 Context 门禁：尚未开始。

### 明确不做

- 不在当前阶段建设通用向量知识库；
- 不建设自动记忆平台；
- 不批量生成未经验证的 Skills；
- 不进入无人值守 Agent 编排；
- 不将自应用结果直接视为跨项目稳定结论。

| 平台适配尚未验证 | 不同 Agent 读取方式可能不同 | 保持核心标准平台无关 | v0.3 |
