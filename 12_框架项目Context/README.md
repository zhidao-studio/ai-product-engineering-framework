## 1. 项目身份

- **仓库**：`zhidao-studio/ai-product-engineering-framework`；
- **一句话定位**：由 zhidao-studio 专有维护、跨平台、可验证的 AI 产品工程框架；

## 2. 当前目标与退出标准

### 当前目标

将 v0.1.x 的 Context 原则转化为可执行的事实源、项目 Pack、阶段 Context、任务 Pack、装配、冲突、回写、检查清单和候选模板，并通过经授权的正式真实业务参考工程验证。

### 里程碑 A 退出标准

一个经授权的正式真实业务任务能够证明：新的人员或 Agent 不依赖历史会话，也能恢复项目背景、识别权威事实、发现冲突、明确范围，完成三层验证并回写结果；同时能够记录 Context 填写成本、人工修正、遗漏和任务成本。

### 当前验证位置

- Framework 自应用：**已通过并由项目维护者确认**；
- 自应用结果：P0 全通过，Context 成熟度 93/100；
- v0.1.1 全量复核：**已完成并提交**；
- v0.1.2 专有许可与阶段清场：**已完成并提交**；
- v0.1.3 业务验证事实同步与版本治理：**已完成并发布**；
- v0.1.4 术语易懂化与表达统一：**已完成并发布**；
- YouYu PR #1：**初步工程证据**，不计为正式业务参考任务；
- YouYu 正式业务参考工程验证：`not_started`；
- 完整前后端业务闭环：尚未开始；
- 自动 Context 与 Harness 检查关卡：尚未开始。

### 明确不做

- 不在当前里程碑建设通用向量知识库；
- 不建设自动记忆平台；
- 不批量生成未经验证的 Skills；
- 不进入无人值守 Agent 编排；
- 不将 Framework 自应用或初步工程证据视为真实业务参考工程通过；
- 不将仓库内容提供给未授权的外部个人、模型、知识库或第三方服务。
