## 2. 阶段目标

把现有 Harness 抽象定义转化为能够指导后续实现的控制基线，明确：

1. 风险等级和相应控制强度；
2. 生命周期阶段进入、退出和退回条件；
3. 用户可见变更、高保真和工程实现之间的路线判断；
4. AI 自主执行、人工批准和失败停止的责任边界；
5. 后续任务边界、依赖、一致性检查和证据模板的统一上游规则。

### 本阶段不解决

- 不批量生成 Harness 模板；
- 不建设多 Agent、Skills 或 Loop 自动编排；
- 不创建可视化工作台或通用 CI 平台；
- 不修改 YouYu 业务代码；
- 不升级稳定版本或资产成熟度。

## 4. 阶段内工作边界

### 允许开展

- 完善 Harness 六类控制面和最小执行模型；
- 建立风险等级、人工确认和自动执行强度；
- 建立阶段进入、退出、阻塞和退回判断；
- 同步 Framework 当前状态、Roadmap、README 和未发布记录。

### 禁止提前开展

- 未经 B1 规则约束即批量编写自动检查脚本；
- 将候选检查写成已经在参考工程执行通过；
- 启动 Skills、Agents 与 Loop 里程碑 C；
- 改变三平面、十阶段、五大基础设施、人类最终责任或许可模式。

### 人工确认点

- 高风险和严重风险控制强度的放宽；
- 用户可见范围和高保真批准；
- 敏感数据、权限、生产配置、发布和回滚；
- B1 退出和 B2 启动。
