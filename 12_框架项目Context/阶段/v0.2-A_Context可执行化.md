## 2. 阶段目标

把“上下文必须外置”从原则转化为可执行机制，使项目能够：

- 维护单一事实源和统一状态；
- 通过项目、阶段和任务 Context 恢复认知；
- 按任务裁剪最小 Context；
- 发现冲突、过期、缺失和敏感信息；
- 固定执行基线；
- 将决策、证据和经验回写长期资产；
- 为 Harness 门禁提供明确输入。

## 3. 本阶段不做

- 不建设向量数据库和通用知识平台；
- 不建设自动记忆产品；
- 不直接进入完整 Skill 标准化；
- 不宣布候选模板已经跨项目稳定；
- 不以 Framework 自应用替代业务参考工程验证。

## 8. 退回路径

- 若 Context 模型无法表达项目事实，退回全局模型和 DEC-006；
- 若模板过重或遗漏，退回对应规范和候选模板；
- 若任务无法控制修改范围，进入 Harness 设计，但不得用 Harness 掩盖 Context 缺失；
- 若业务目标或高保真不清，退回产品定义或体验设计阶段。
