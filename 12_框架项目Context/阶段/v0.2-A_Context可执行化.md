## 2. 阶段目标

把“上下文必须外置”从原则转化为可执行机制，使项目能够：

- 维护单一事实源和统一状态；
- 通过项目、阶段和任务 Context 恢复认知；
- 按任务裁剪最小 Context；
- 发现冲突、过期、缺失和敏感信息；
- 固定执行基线；
- 将决策、证据和经验回写长期资产；
- 为修改边界、检查关卡和人工确认提供明确输入；
- 在正式真实业务任务中证明模板有效且成本可接受。

## 4. 本阶段不做

- 不建设向量数据库和通用知识平台；
- 不建设自动记忆产品；
- 不直接进入完整 Skill 标准化；
- 不宣布候选模板或数据库规范已经单项目、跨项目或稳定；
- 不以 Framework 自应用、初步工程证据、基础构建或 HTML 原型替代正式业务参考工程验证；
- 不在 A2 正式业务实现任务尚未建立时提前宣布进入 Harness B；
- 不将专有仓库内容发送到未授权模型、知识库、数据集或第三方系统。

## 8. 里程碑 A 退出标准

只有同时满足以下条件，维护者才能批准退出里程碑 A：

1. YouYu 完成一个小而完整的正式业务切片；
2. 项目、阶段和任务 Context 能支持新 Agent 独立恢复任务；
3. 单一事实源、冲突、缺失、过期和回写机制在真实任务中生效；
4. 静态、运行和模拟用户三层验证完整；
5. Context 填写成本、人工修正、遗漏和冲突已记录；
6. 参考工程经验已回写 Framework；
7. 维护者明确批准退出。
