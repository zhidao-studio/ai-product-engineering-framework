## 2. 阶段目标

把“上下文必须外置”从原则转化为可执行机制，使项目能够：

- 维护单一事实源和统一状态；
- 通过项目、阶段和任务 Context 恢复认知；
- 按任务裁剪最小 Context；
- 发现冲突、过期、缺失和敏感信息；
- 固定执行基线；
- 将决策、证据和经验回写长期资产；
- 为 Harness 检查关卡提供明确输入；
- 在专有许可和授权工具范围内安全处理 Context。

## 3. 本阶段不做

- 不建设向量数据库和通用知识平台；
- 不建设自动记忆产品；
- 不直接进入完整 Skill 标准化；
- 不宣布候选模板已经跨项目稳定；
- 不以 Framework 自应用或初步工程证据替代正式业务参考工程验证；
- 不将专有仓库内容发送到未授权模型、知识库、数据集或第三方系统。

## 8. 退回路径

- 若 Context 模型无法表达项目事实，退回全局模型和 DEC-006；
- 若模板过重或遗漏，退回对应规范和候选模板；
- 若任务无法控制修改范围，进入 Harness 设计，但不得用 Harness 掩盖 Context 缺失；
- 若人工检查关卡无法约束实际合并行为，停止成熟度升级并优先修复 Harness；
- 若业务目标或高保真不清，退回产品定义或体验设计阶段；
- 若授权、许可或数据边界不清，停止任务并回到 DEC-009、LICENSE 和人工授权。
