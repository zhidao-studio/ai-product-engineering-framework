## 1. 进入依据

- 里程碑 B 已以 `conditional_pass` 有限退出；

## 2. 首个封装目标

首个候选能力：

```text
中文名称：受控任务验证 Skill
兼容标识：controlled-task-validation
```

它只封装已经验证的最小闭环：

```text
读取任务Context
→ 确认风险和人工责任
→ 检查允许、禁止与高风险路径
→ 检查证据结构和有限结论
→ 输出通过、阻塞或需要人工确认
```

## 3. 不纳入首个 Skill

- 依赖变更评估；
- OpenAPI、Schema、服务端和客户端全链一致性；
- 完整安全审计；
- 自动回滚和生产发布；
- 多 Agent 编排；
- 跨平台特有工具自动适配。

这些能力仍为 `candidate`，不得通过 Skill 名称暗示已经成熟。

## 4. C1 输出

- Skill 触发与不适用条件；
- 必需输入和事实源；
- 执行步骤与停止条件；
- 标准输出和人工确认点；
- 复用现有脚本、模板和参考证据的方式；
- Framework 自应用和 YouYu 下一任务的验证计划；
- Agent 与 Loop 的最小交接边界。
