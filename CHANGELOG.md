# 变更记录

本文件记录 AI 产品工程框架的重要版本变化。

## [未发布：0.2.0]

目标：将 Context 与 Harness 从框架定义转化为可执行、可验证的工程能力。

### Context 可执行化

- 确立“框架级、项目级、阶段级、任务级”四级 Context 作用域与独立决策经验回写层。
- 新增 Context 工程总览图和模块入口。
- 完善 Context 与记忆管理总览，区分权威事实源、Context Pack、模型会话和自动记忆。
- 新增事实源与状态规范，定义元数据、状态、版本、新鲜度、敏感级别和权威索引。
- 新增项目 Context Pack、阶段 Context、任务 Context Pack 规范。
- 新增 Context 装配、预算、冲突、缺失、过期和敏感信息处理规范。
- 新增决策与经验回写规范和 Context 完整性检查清单。
- 提供项目、阶段、任务、冲突和经验回写五类候选模板。
- 统一 Context 机器状态枚举、中文显示和 Pack 版本字段。
- 修正任务 Context 示例的生命周期阶段映射。

### Framework 自应用验证

- 建立 `12_框架项目Context`，保存 Framework 自身的项目、阶段、任务和检查记录。
- 使用候选模板建立 Framework 项目 Context Pack、里程碑 A 阶段 Context 和任务 Context Pack。
- 完成 CTX-CHECK-001，自应用 Context 基线 P0 全部通过，成熟度评分 93/100。
- 项目维护者已经确认自应用结果。
- 自应用只证明仓库记忆和一致性管理，不能替代真实业务产品验证。

### 当前开发状态

- YouYu 首轮业务参考任务已经形成项目、阶段和任务 Context、修改边界以及 GATE-001 至 GATE-004 的真实证据。
- GATE-005 签名真机模拟用户验收、P0/P1 体验问题关闭和 Context 成本复盘仍未完成。
- Context 模板成熟度继续为 `candidate`。
- 尚未建设自动 Context 装配器、向量知识库、版本门禁或完整 Harness 脚本。
- 尚未完成包含前端、后端、OpenAPI 和 Schema 的完整端到端业务验证。

## [0.1.2] - 2026-07-12

版本定位：在 v0.1.1 稳定治理基线上增加专有闭源许可、外部使用边界和进入真实业务验证前的阶段清场，不改变框架宪法、三平面模型、十阶段生命周期和五大基础设施。

### 专有许可

- 新增根目录 `LICENSE`，采用 Proprietary / All Rights Reserved。
- 明确本项目不是开源项目，禁止未经授权使用、复制、修改、分发、商业化、模型训练或创建衍生作品。
- 新增 DEC-009，固定专有许可、书面授权和仓库私有化要求。
- 明确 Public 可见性不等于授予使用权，访问控制需要将仓库改为 Private。

### 文档与协作治理

- README 将项目定位调整为“由 zhidao-studio 专有维护、跨平台、可验证”。
- AGENTS 增加 LICENSE 读取顺序、知识产权边界和未授权外发禁止规则。
- CONTRIBUTING 改为仅限受邀协作者，不再接受未经邀请的外部贡献。
- 愿景定位和边界声明删除“开放标准”歧义，加入专有权利定位。
- 新增 `v0.1.2专有许可与阶段清场报告.md`。

### 版本边界

- 当前稳定版本升级为 v0.1.2。
- v0.2.0 / A 的 Context 规范、候选模板和业务验证仍属于未发布开发内容。
- 本修订不代表 Context、Harness、Skills 或多平台适配已经稳定。
