# 模板资产

本目录将存放把框架标准转化为可执行输入和输出的模板。

## 计划中的模板族

| 生命周期或基础设施 | 计划模板 |
|---|---|
| 价值验证 | 产品机会评估、用户问题验证、成功指标 |
| 产品定义 | PRD、不做清单、业务规则、验收断言 |
| 体验与高保真 | 用户流程、页面清单、状态矩阵、高保真确认记录 |
| 工程规格 | 架构、API、数据库、前端、后端、移动端规格 |
| Context | 项目记忆包、任务上下文包、设计决策 |
| Harness | 修改边界、阶段门禁、契约检查、发布检查 |
| Skills 与 Agents | Skill 定义、角色输入输出契约、编排清单 |
| 验证 | 代码审查、运行验证、模拟用户验收、验证报告 |
| Loop | 反馈分类、指标复盘、失败归因、改进记录 |

## v0.1 约束

v0.1 只定义模板分类和边界，不批量生成未经真实项目验证的空模板。

模板进入正式目录前必须：

1. 在至少一个参考工程中使用；
2. 证明能够减少遗漏或歧义；
3. 明确必填项与可裁剪项；
4. 有清晰示例和反例；
5. 能被人和 AI 共同理解。
