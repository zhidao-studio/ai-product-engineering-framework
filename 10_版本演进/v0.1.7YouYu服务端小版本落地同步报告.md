## 1. 版本目的

v0.1.7 不新增 Framework 核心模型，而是记录第一个真实参考工程从“候选工程规格”进入“服务端代码已实现、等待真实验证”的阶段变化。

本版本重点验证 Framework 是否能够做到：

```text
产品与体验事实
→ 工程差异和引用扫描
→ 人工参数确认
→ 数据库与OpenAPI
→ 受控代码实现
→ 独立验证任务
```

## 2. YouYu v0.1.1 已落地内容

### 数据库

- `app_user_account`；
- `app_user_profile`；
- `auth_sms_code`；
- `user_agreement_confirmation`；
- 32 位字符串主键和关联 ID；
- 七个基础字段；
- Y/N 有效与删除标记；
- 数据库触发器维护创建和修改时间；
- 独立回滚脚本；
- 不改写历史管理员 `sys_user`、角色和权限表。

### 接口与实现

- 手机号验证码发送和统一登录注册；
- 账号自动创建和基础资料创建；
- 协议版本确认；
- 当前用户、只读账号信息、昵称和简介更新；
- 7 天 Bearer 会话；
- Gateway 精确公开路径；
- 字符串 App 身份和旧管理员数值身份兼容隔离；
- HMAC 手机号定位、AES-GCM 密文和脱敏展示；
- 验证码过期、重发、频控、失败锁定、一次使用和旧码作废。

### 失败关闭

- 真实短信供应商未接入时生产模式不可发送；
- 安全密钥缺失时手机号与验证码接口失败；
- 协议版本或 HTTPS 内容地址缺失时登录与协议查询失败；
- 头像和地区能力未接入时非空更新失败；
- 不将未实现能力伪装为成功。

## 3. Framework 得到的验证价值

本轮证明了以下框架资产能够实际参与代码落地：

- 项目 Context 能保存阶段、任务、阻塞和证据状态；
- 工程差异清单阻止直接改造错误的旧账号模型；
- 数据库候选规范约束新表主键、基础字段和触发器；
- 人工确认点阻止 AI 自行决定验证码、会话和个人资料参数；
- 接口和数据约定先于代码形成；
- 任务边界保持管理员认证、App 账号、iOS 和后续模块隔离；
- 版本和 Context 能明确区分 `implemented` 与 `validated`。

## 4. 尚不能得出的结论

本轮不能证明：

- SQL 已在 MySQL 真实执行；
- Maven 测试或打包已经通过；
- Gateway、App、数据库和共享会话已经联通；
- 验证码安全、并发和限流在运行中正确；
- iOS 已实现或符合高保真；
- 产品具备生产发布条件；
- Context 模板或数据库规范已经达到 `single_project_validated`。
