## 1. 版本定位

v0.1.9 是 Framework 的全仓事实源收敛、YouYu v0.1.3 静态修复同步和发布治理修订版本。它不改变框架核心模型，不代表 YouYu 运行验证完成，也不代表 v0.2.0 里程碑 A 已退出。

## 2. YouYu v0.1.3 已修复

### 数据库审计

- 保留 v0.1.1 历史建表迁移；
- 新增 v0.1.3 触发器修订和安全回滚；
- 插入时无条件覆盖创建与修改时间；
- 更新时保护历史创建时间并刷新修改时间；
- 触发器名称统一为 `trg_<table>_before_insert/before_update`；
- 数据库检查验证触发器正文，而非只统计数量；
- 主键、关联 ID、审计列、每个唯一索引、所属表和复合字段顺序逐项断言。

### 验证码并发与安全

- 验证码读取使用 `SELECT ... FOR UPDATE`；
- 错误次数在数据库事务锁内读取和按主键更新；
- 第五次错误固定为 `LOCKED:5`；
- 同一验证码并发正确请求只能有一个消费成功；
- 已使用、已锁定、已作废和已过期验证码不能再次登录；
- 协议缺失等后续业务失败回滚验证码占用；
- `SmsAttemptException` 不回滚边界没有扩展到其他业务异常。

### 配置、契约与输入

- App/Admin 不再默认激活 dev；
- 数据库账号密码不再使用仓库内 `root/root` 默认值；
- 公共配置移除 MyBatis 完整 SQL 参数输出；
- Gateway 下游地址支持环境变量；
- 协议确认数组最多两项，空对象、重复对象和同类型冲突版本失败；
- 昵称和简介长度统一按 Unicode 码点计算；
- Trace ID 限制为 32 位小写十六进制；
- 短信适配边界区分规范化发送号码和脱敏审计号码；
- 网络来源频控明确为 `not_implemented`，由 YouYu TASK-012 管理。

### Context 与验证链

- YouYu 项目 Context 采用 `project_context_id`、Pack 版本、稳定版本、目标版本、基线和来源提交等 Framework 字段；
- TASK-011 补齐任务 Context Pack 字段并保持阻塞；
- 产品规则只保存稳定规则，不再保存动态实现状态；
- 高保真明确服务端已实现但未运行验证，iOS 仍未开始；
- OpenAPI v0.1.3 成为当前唯一权威契约；
- 新增本地可执行 `scripts/check-account-slice.sh`；
- GitHub Actions 调用同一脚本，不再维护平行验证逻辑；
- 新增 SERVER-CHECK-003，静态复核 `passed`，运行执行 `blocked_external_runtime_evidence`。
