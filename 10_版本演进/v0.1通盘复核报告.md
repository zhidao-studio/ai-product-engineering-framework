# v0.1 通盘复核报告

- 复核日期：2026-07-12
- 复核范围：README、AGENTS、框架宪法、全局模型、角色、Context、Harness、Skills 与 Agent、Loop、模板、参考工程、Roadmap、设计决策和图片资产
- 复核结论：**方向成立，v0.1 已具备进入逐模块可执行化阶段的基础；进入下一阶段前需先收敛事实源和统一生命周期口径。**

## 1. 总体评价

v0.1 已经明显区别于 Prompt 集合、AI Coding 教程和 Skills 收藏库，形成了完整的上层框架：

1. 产品价值生命周期负责回答产品如何从机会走向持续价值；
2. Context、Harness、Skills、Agents、Loop 作为横向基础设施贯穿全部阶段；
3. 治理与组织记忆负责责任、决策、权限、指标、成本、变更和长期演进；
4. 高保真预览、模拟用户验收和持续反馈已经进入正式流程；
5. README、AGENTS.md、专题文档和设计决策之间形成了初步记忆体系。

这套结构可以支撑后续逐模块实现，不需要再次推翻顶层模型。

## 2. 已验证的优势

### 2.1 框架定位清楚

项目明确不是单一模型、IDE、Prompt、Skill 或全自动软件工厂，而是跨平台的 AI 产品工程方法和资产体系。

### 2.2 生命周期与 AI 能力分离

三平面模型避免把 Context、Harness 和 Loop 错误地当成一次性生命周期阶段，是当前框架最关键的结构性判断。

### 2.3 人类责任没有被自动化口号稀释

价值判断、高保真确认、高风险审批、验收豁免、发布和回滚仍由明确的人类角色负责。

### 2.4 Context 被提升为组织记忆

仓库事实源、设计决策、任务上下文和经验回写共同解决跨会话、跨模型和长期任务中的失忆问题。

### 2.5 验证不是单一测试

静态与契约验证、运行验证、模拟用户验收形成三层证据链，能够发现“代码正确但产品不可用”的问题。

### 2.6 Loop 有停止和归因机制

Loop 不被描述为无限自我进化，而是包含目标、指标、重试预算、停止条件、人工升级和长期资产回写。

## 3. 复核发现与修正

### P0-1：出现平行事实源

发现：

- `02_全局模型` 与新增的 `02_核心模型` 同时存在；
- `11_设计决策` 与新增的 `05_设计决策记录` 同时存在；
- `适用场景与期望.md` 与 `AI产品工程适用场景.md` 内容重叠。

风险：AI 和维护者无法确定哪个是当前权威来源，违背 Context 单一事实源原则。

修正：

- 保留 `02_全局模型`；
- 保留 `11_设计决策`；
- 保留并扩充 `适用场景与期望.md`；
- 删除平行目录和重复文档；
- 在 AGENTS.md 中禁止新增同义平行目录。

### P0-2：生命周期存在九阶段和十阶段两种口径

发现：部分文档合并“模拟用户验收与发布”，部分文档将其拆分。

风险：验收责任和生产发布责任混淆，后续无法分别建设验收门禁与发布门禁。

修正：

- 新增 `DEC-005`；
- 统一为十阶段生命周期；
- 同步 README、AGENTS.md、全局模型和边界声明。

### P0-3：概念图片引用断链

发现：核心模型文档引用 `AI产品工程框架v0.1概念总览.png`，但文件未实际存在。

修正：将 ChatGPT 生成的 PNG 正确提交到仓库，并明确图片用于认知、正式定义以 Markdown 和 Mermaid 为准。

### P1-1：宪法层导航不够显式

发现：README 只提供一个“框架定义”入口，没有分别展示愿景、场景、原则和边界。

修正：README 增加四份宪法文档的独立导航和职责说明。

### P1-2：Context 模型描述尚未可执行

发现：现有 Context 文档已经定义原则和层级，但尚缺项目 Context Pack、任务 Context Pack、装配规则、冲突处理、状态管理和候选模板。

处理：列为 v0.2 第一子阶段。

### P1-3：模板、门禁和参考工程尚未形成验证链

这是 v0.1 的合理边界，不视为缺陷。后续必须遵循“真实使用 → 验证 → 再稳定化”的顺序。

## 4. 当前一致性基线

### 权威目录

```text

02_全局模型      三平面模型与十阶段生命周期
03_角色体系      人类与 AI 的职责和协作
04_Context工程   项目记忆与任务上下文
05_Harness工程   执行控制与门禁
06_Skills与Agent 能力封装与角色协作
07_Loop工程      持续反馈与演进
08_模板资产      可执行模板入口

10_版本演进      Roadmap 与复核报告
11_设计决策      重要取舍和变更原因
```

### 十阶段生命周期

```text
战略与价值验证
→ 产品定义
→ 用户体验设计
→ 高保真原型预览与确认
→ 工程规格设计
→ 受控任务执行
→ 质量与安全验证
→ 模拟用户验收
→ 发布交付
→ 运行反馈与持续迭代
```

### 五大横向基础设施

```text
Context：让 AI 理解
Harness：让 AI 受控
Skills：让 AI 会做
Agents：让 AI 协作
Loop：让系统闭环
```

## 6. 下一阶段决定

v0.2 不同时展开所有模块，按依赖顺序推进：

1. **Context 可执行化**：先保证 AI 获得正确、必要、可追溯的信息；
2. **Harness 可执行化**：再建立任务边界、契约和门禁；
3. **真实任务验证**：用参考工程验证 Context 与 Harness；
4. **Skills 候选化**：只把已经验证的方法封装为 Skill。

第一子阶段的完成标准是：一个真实任务可以通过项目 Context Pack 和任务 Context Pack恢复背景、识别权威事实、发现冲突、明确范围并知道结果应回写到哪里。
