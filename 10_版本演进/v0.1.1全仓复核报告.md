# v0.1.1 全仓复核报告

- 复核日期：2026-07-12
- 复核对象：`main` 分支当前全部框架内容
- 目标版本：v0.1.1
- 当前建设阶段：v0.2-A Context 可执行化
- 复核结论：**v0.1.1 可以作为 v0.1.0 的治理修订版本发布；v0.2-A 内容继续保持未发布和候选状态。**

## 1. 复核范围

本次复核覆盖：

- `README.md`、`AGENTS.md`、`CHANGELOG.md` 和 `VERSION`；
- 框架宪法：愿景、适用场景、核心原则和边界；
- 三平面全局模型与十阶段产品价值生命周期；
- 人类与 AI 角色体系；
- Context、Harness、Skills、Agents 和 Loop；
- 模板资产和参考工程要求；
- Roadmap、版本规范和设计决策；
- Framework 自身项目 Context、阶段、任务和验证记录；
- Mermaid、SVG、PNG 和内部导航。

## 2. 总体判断

### 2.1 v0.1 核心模型稳定

以下结论已形成一致事实源，不需要在当前版本继续调整：

1. 框架采用“产品价值生命周期、AI 工程基础设施、治理与组织记忆”三平面模型；
2. 产品价值生命周期统一为十个阶段；
3. Context、Harness、Skills、Agents、Loop 是贯穿生命周期的横向基础设施；
4. 高保真原型预览、模拟用户验收、发布交付和持续反馈均为正式环节；
5. 人承担价值、高风险审批、最终验收和发布责任；
6. Framework 必须通过真实参考工程验证，不能以文档数量证明成熟。

### 2.2 v0.1.1 属于兼容修订

v0.1.1 没有推翻框架宪法和三平面模型，主要完成：

- 全仓通盘复核；
- 重复事实源和目录收敛；
- 生命周期口径统一；
- 图片、链接和导航修复；
- 稳定版本、开发阶段和 Context Pack 版本分离；
- 建立版本单一事实源和版本管理规范；
- 修复重复设计决策编号。

因此适合增加修订版本，由 v0.1.0 升级为 v0.1.1。

## 4. 本轮发现与处理

### P0-1：稳定版本仍标记为 v0.1.0

**发现：** v0.1 全仓 Review 和治理修订已经完成，但 README、AGENTS 和项目 Context 仍使用 v0.1.0。

**处理：**

- 新增根目录 `VERSION`，设置为 `0.1.1`；
- 将 README、AGENTS、CHANGELOG、Roadmap 和项目 Context 统一到 v0.1.1；
- v0.2-A 保留为当前建设阶段，不视为稳定发布。

### P0-2：CHANGELOG 混合两个版本范围

**发现：** “未发布”中同时包含 v0.1 Review 修订和 v0.2-A Context 新能力。

**处理：**

- 将 Review、事实源收敛、十阶段统一、图片导航修复和版本治理移动到 v0.1.1；
- 将 Context 可执行化、自应用验证和候选模板保留在“未发布”。

### P0-3：设计决策编号重复

**发现：** 在版本治理过程中出现两份 `DEC-008`。

**处理：**

- 保留 `DEC-008_采用语义化发布版本与独立开发里程碑.md`；
- 删除重复决策文件；
- 设计决策索引继续保持唯一编号。

### P1-1：Context 自应用状态存在过期描述

**发现：** 阶段和任务 Context 仍写“自应用进行中”或“等待人工确认”，与已经完成的 93/100 检查不一致。

**处理：**

- 将自应用任务标记为完成；
- 将 Framework 自应用标记为通过；
- 保留业务参考工程验证为 v0.2-A 的退出阻塞项。

### P1-2：版本事实源此前不够明确

**处理：**

```text
VERSION
→ CHANGELOG
→ Roadmap
→ 版本管理规范
→ README / AGENTS / 项目 Context 摘要
```

## 6. v0.1.1 发布范围

v0.1.1 稳定发布包含：

- 框架宪法和全局模型的 Review 修订；
- 单一事实源治理；
- 十阶段生命周期统一；
- 图片和导航修复；
- README、AGENTS 和版本摘要一致性修订；
- 版本管理规范；
- 全仓复核报告和版本决策。

以下内容存在于 `main`，但仍属于 v0.2.0 未发布开发内容：

- Context 可执行化规范；
- Context 完整性检查清单；
- Context 候选模板；
- Framework Context 自应用验证；
- 后续 Harness 可执行化准备。

## 7. 剩余风险

- 尚无自动 Markdown 链接、Mermaid、元数据和版本一致性门禁；
- Context 候选模板尚未经过真实业务参考工程验证；
- Harness 仍以规范为主，尚未形成可运行门禁脚本；
- `main` 同时承载稳定版本和下一版本开发内容，需要持续明确成熟度；
- 不同 AI 平台的实际 Context 加载和 Agent 行为尚未比较验证。

## 8. 最终结论

**批准将当前稳定版本从 v0.1.0 修订为 v0.1.1。**

v0.1.1 是治理和一致性修订，不代表 v0.2 Context 与 Harness 能力已经稳定。下一步继续推进 v0.2-A 的真实业务参考工程验证，并交叉开始 v0.2-B Harness 可执行化。
