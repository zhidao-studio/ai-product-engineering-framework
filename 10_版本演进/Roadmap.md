## v0.2.0：Context 与 Harness 可执行化

目标：让一次真实 AI 任务获得正确 Context，在明确边界和检查关卡下完成，并留下可验证、可回写的证据。

### 里程碑 A：Context 可执行化

#### A1：规范、候选模板与 Framework 自应用

状态：`completed`

已完成：

- 框架、项目、阶段和任务四级 Context；
- 独立决策与经验回写层；
- 权威事实源、状态、版本、新鲜度和敏感级别；
- Context 装配、裁剪、预算、冲突和安全规则；
- 完整性检查和候选模板；
- Framework 自应用基线。

A1 只证明规范和自应用成立，不代表候选模板已经通过真实业务验证。

#### A2：正式业务参考工程验证

状态：`active`

当前子步骤：**YouYu v0.1.2 运行验证待执行**。

已完成：

- 产品范围、业务规则、体验和高保真候选；
- 工程差异、旧代码与 SQL 引用扫描；
- App 用户与管理员账号域隔离；
- 数据库基础规范与关键参数确认；
- 四张账号域表迁移、触发器与回滚；
- v0.1.2 OpenAPI、错误码和字段约定；
- 手机号验证码登录注册服务端实现；
- 字符串身份、Bearer 网关和共享 Redis 配置；
- Sa-Token 编译、验证码事务、主键更新、数据源、Mapper 扫描和契约缺陷修复；
- 单元测试、数据库断言和联合验证工作流；
- SERVER-CHECK-002 与 TASK-011 阻塞状态。

当前必须取得：

1. Maven 测试与打包成功日志；
2. MySQL 迁移、结构、触发器、回滚和重复迁移成功日志；
3. Gateway、App、MySQL、Redis 和内部身份联合运行；
4. 验证码锁定、协议回滚、登录、资料和退出场景；
5. 日志与敏感信息检查；
6. Redis 故障和配置不一致验证；
7. 高保真逐页批准；
8. iOS 网络、Keychain、页面和真机验收；
9. 模拟用户端到端验收；
10. 失败、成本、人工修正和 Framework 改进回写；
11. 人类责任人批准里程碑结论。

状态边界：

- YouYu 服务端实现为 `implemented_not_runtime_validated`；
- 静态复核为 `conditional_pass`；
- 自动检查为 `defined_not_executed`；
- YouYu 正式业务验证为 `not_started`；
- Context 模板和数据库规范保持 `candidate`；
- Harness B 尚未开始；
- GitHub Actions 无运行状态时不得声称远程成功或失败。

A2 退出条件：

- 真实业务切片完成静态、数据库、运行、安全和模拟用户验收；
- 项目、阶段和任务 Context 可被新 Agent 独立恢复；
- 修改边界和检查关卡在实际任务中生效；
- 失败、人工修正、成本和遗漏有记录；
- Framework 改进完成回写；
- 人类责任人批准。

### 里程碑 B：Harness 可执行化

状态：`not_started`

计划能力：

- 阶段检查关卡；

### 里程碑 C：Skills、Agents 与 Loop

状态：`not_started`

计划能力：

- 将已验证方法封装为 Skills；
- 建立角色化 Agents 和受控交接；
- 建立任务、阶段、产品和框架四级反馈闭环；
- 建立失败归因、成本、指标和资产成熟度管理。

启动条件：Context 和 Harness 已在真实任务中稳定运行。
