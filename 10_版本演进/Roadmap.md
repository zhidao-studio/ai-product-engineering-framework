# Roadmap

> 路线图按“先验证方法，再封装能力，最后自动编排”的顺序推进。发布版本采用语义化版本，`A/B/C` 只表示次版本内部开发里程碑；`A1/A2` 只用于说明里程碑 A 的工作段，不是独立版本或里程碑。

## v0.2.0：Context 与 Harness 可执行化（开发中）

目标：让一次真实 AI 任务能够获得正确 Context，在明确边界和检查关卡下完成，并留下可验证、可回写的证据。

### 里程碑 A：Context 可执行化（当前）

#### A1：规范、候选模板与 Framework 自应用（已完成）

已完成：

- 框架、项目、阶段、任务四级 Context 作用域；
- 独立决策与经验回写层；
- 权威事实源、状态、版本、新鲜度和敏感级别；
- 项目、阶段和任务 Context Pack；
- Context 装配、裁剪、预算和安全规则；
- 冲突、缺失、过期、决策、证据和经验回写；
- Context 完整性检查清单和第一批候选模板；
- Framework 自应用基线检查，93/100，人工确认通过；
- YouYu PR #1 的初步工程证据已经回写，用于发现 Xcode 注册、资源、真机架构、Debug 路由和合并控制问题。

A1 完成只证明规范和自应用成立，不代表模板已经通过正式业务项目验证。

#### A2：正式业务参考工程验证准备（当前，active）

当前子步骤：**YouYu 业务功能准备**。

已完成：

- YouYu 正常完整产品工程目录已经建立；
- YouYu 已建立项目 Context、工程设计、接口与数据约定、检查关卡、测试验收和部署运维入口；
- `YouYu-iOS/` 已确认为唯一正式 iOS 工程，并正式放弃 mPaaS；
- YouYu 已完成第一轮静态工程复核；
- iOS 已完成 CocoaPods 恢复、真机 Debug 构建、安装与启动；临时 ngrok 地址已移除，服务地址改为构建配置注入；
- 服务端已完成 Maven 多模块构建、App 直连和网关基础接口验证；内部身份传递已增加短时签名与单元测试；
- 采集工程已完成 Spring Boot 3 依赖修复和构建；
- GitHub Actions 已通过服务端、采集工程和 iOS 的远程重复构建检查；
- 工程基础当前结论为 `conditional_pass`，YouYu 已进入业务功能准备。

当前阻塞：

- Git 历史敏感信息尚未轮换、清理和建立扫描检查关卡；
- Gateway、认证服务和下游服务共享会话存储，以及下游网络隔离尚未证明；
- 采集工程尚未建立“候选数据 → 人工审核 → 正式钓点数据”的待审入库运行控制；
- iOS 测试/生产环境确认、真机人工视觉验收和网络异常验收尚未完成。

状态边界：

- YouYu 正式业务参考工程验证状态仍为 `not_started`；
- 工程基础修复与业务准备不等于正式业务参考任务已经开始；
- 可以开始产品、体验和接口约定整理；正式业务代码实现继续为 `blocked`；
- Context 模板继续保持 `candidate`；
- Harness 里程碑 B 尚未正式开始。

仍需完成：

- 由维护者确认首个业务功能的用户问题、范围、不做清单和验收断言；
- 完成用户流程、页面状态、交互说明和高保真人工确认；
- 建立接口与数据约定的唯一权威来源；
- 并行关闭或形成可审计隔离结论：历史敏感信息、共享会话与网络隔离、采集待审入库和 iOS 真机人工验收；
- 重新建立 YouYu 正式项目 Context、阶段 Context 和任务 Context；
- 配置修改边界、检查关卡和人工确认节点；
- 完成静态、运行和模拟用户三层验证；
- 记录 Context Pack 填写成本、人工修正、遗漏、冲突和任务成本；
- 根据完整业务验证修正规范和模板；
- 判断模板是否达到 `single_project_validated`；
- 由项目维护者批准里程碑 A 退出。

里程碑 A 退出标准：新的人员或 Agent 不依赖历史会话，也能在经授权的正式真实业务任务中恢复背景、识别权威事实、发现冲突、明确修改范围，完成三层验证并回写结果；同时有足够证据证明模板成本与任务风险匹配。

### 里程碑 B：Harness 可执行化（尚未正式开始）

- 阶段进入与退出检查关卡；
- 文件和模块修改边界；
- OpenAPI、Schema、依赖和功能行为约定一致性检查；
- 权限、安全、成本和停止条件；
- 三层验证模板；
- 模拟用户验收与发布前检查关卡分离；
- 变更、发布和回滚清单；
- Markdown 链接、Mermaid、版本、状态、许可证和 Context 元数据检查；
- 第一套可运行检查关卡脚本；
- PR 任务状态、Required Checks、人工批准和实际合并权限的一致性控制；
- Xcode 源码注册、资源注册、真机架构和运行路由候选检查关卡。

退出标准：在参考工程中证明检查关卡能够发现真实问题，并且不会因为过度规则阻止正常任务；文档中的人工检查关卡必须能够约束实际合并行为，而不是仅作为说明存在。

- 覆盖前端、后端、OpenAPI、Schema、依赖和联调；
