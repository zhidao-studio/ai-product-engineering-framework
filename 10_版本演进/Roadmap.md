## v0.2.0：Context 与 Harness 可执行化

目标：让一次真实 AI 任务获得正确 Context，在明确边界和检查关卡下完成，并留下可验证、可回写的证据。

### 里程碑 A：Context 可执行化

#### A1：规范、候选模板与 Framework 自应用

状态：`completed`

已完成四级 Context、独立回写层、事实源与状态、装配与裁剪、冲突与安全、完整性检查、候选模板和 Framework 自应用。A1 不证明候选资产已通过真实业务验证。

#### A2：正式业务参考工程验证

状态：`active`

当前子步骤：**YouYu v0.1.4 运行验证**。

已完成：

- 产品规则、体验定义和高保真候选；
- App 用户与管理员账号域隔离；
- 四张账号域表、历史迁移和 v0.1.3 审计触发器修订；
- 验证码消费行锁、失败次数与单次消费；
- v0.1.4 Redis Lua 原子发送门禁和发送幂等键；
- App 基础数据源、配置启动校验、严格 Bearer 和 Trace 响应头；
- OpenAPI v0.1.4、SERVER-CHECK-004 和可重复本地检查脚本；
- Framework Context 提交真实性和发布一致性检查修订。

当前必须取得：

1. Maven 测试与打包实际成功日志；
2. MySQL 迁移、审计行为、约束和回滚成功日志；
3. Redis Lua 并发发送、幂等和计数实际结果；
4. Gateway、App、MySQL、Redis 和内部身份联合运行；
5. 并发错误 `LOCKED:5`、并发正确单次消费和非法状态失败关闭证据；
6. 协议回滚、资料、Trace 响应头、严格 Bearer、退出和日志安全结果；
7. Redis 停机、网络分区和配置不一致验证；
8. 完整短信 Outbox 和网络来源频控；
9. 高保真逐页批准；
10. iOS、Keychain、真机和模拟用户验收；
11. 失败、成本、人工修正和 Framework 改进回写；
12. 人类责任人批准里程碑结论。

状态边界：

- YouYu 服务端实现：`implemented_not_runtime_validated`；
- 静态复核：`conditional_pass`；
- 运行验证：`blocked_external_runtime_evidence`；
- 正式业务验证：`not_started`；
- 高保真：`draft_for_confirmation`；
- iOS 正式实现：`not_started`；
- Context 模板与数据库规范：`candidate`；
- Harness B：`not_started`；
- 检查定义不得写成运行通过。

A2 退出条件：真实业务切片完成静态、数据库、运行、安全和模拟用户验收；Context 可由新 Agent 独立恢复；修改边界真实生效；失败、人工修正、成本和遗漏有记录；Framework 改进完成回写；人类责任人批准。

### 里程碑 B：Harness 可执行化

状态：`not_started`

计划能力：阶段检查关卡、修改范围和依赖白名单、OpenAPI/Schema/代码一致性、权限与密钥边界、自动检查和人工确认编排、失败停止和证据归档。

启动条件：里程碑 A 完成真实参考工程验证，并确认哪些检查值得标准化。

### 里程碑 C：Skills、Agents 与 Loop

状态：`not_started`

计划能力：将已验证方法封装为 Skills；建立角色化 Agents；建立任务、阶段、产品和框架反馈闭环；管理失败、成本、指标和资产成熟度。
