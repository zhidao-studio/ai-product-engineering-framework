# Roadmap

## v0.1：框架立项

目标：建立完整地图和统一语言。

- 愿景、定位、边界和适用场景；
- 三平面全局模型；
- 产品交付生命周期；
- Context、Harness、Skills、Agents、Loop；
- 人类与 AI 角色；
- 设计决策、模板和参考工程入口；
- Mermaid 与真实图片资产。

退出标准：新参与者能够理解框架结构，后续新增内容都有明确归属。

## v0.2：Harness 可执行化

目标：让一次真实 AI 任务能够被完整控制。

- 项目 Context Pack；
- 标准任务包；
- 阶段、边界、契约和依赖门禁；
- 三层验证模板；
- 变更、发布和回滚清单；
- 第一套门禁脚本。

退出标准：使用参考工程证明门禁能发现真实问题。

## v0.3：Skills 标准化

目标：把已验证方法封装为能力。

- Skill 设计规范；
- 产品定义、设计评审、架构评审、代码审查、用户验收等核心 Skills；
- 触发条件、输入输出、验证和失败升级；
- Claude Code 与 Codex 第一批适配。

退出标准：同一任务在至少两个执行平台上得到结构一致的产物。

## v0.4：参考工程闭环

目标：验证完整生命周期。

- 至少两类真实参考工程；
- 高保真预览到代码实现的视觉验证；
- 用户验收和运行反馈；
- 成本、人工修正比例和任务成功率记录；
- Framework 缺陷清单和修正。

## v0.5：Agent 编排与 Loop

目标：在保持人工关键决策的情况下连接多个角色。

- Agent 输入输出契约；
- 失败分类与升级；
- 任务、阶段、产品和框架四级 Loop；
- 可视化状态和验证报告。

## v1.0：跨平台可验证框架

目标：形成稳定的标准、模板、Skills、门禁、适配和参考实现体系。

v1.0 不以“完全无人化”为成功标准，而以可控、可验证、可迁移和真实产品价值为标准。
