> 路线图按“先验证方法，再封装能力，最后自动编排”的顺序推进。发布版本采用语义化版本，`A/B/C` 只表示次版本内部开发里程碑；`A1/A2` 只用于说明里程碑 A 的工作段，不是独立版本或里程碑。

已完成：

- 框架、项目、阶段、任务四级 Context 作用域；
- 独立决策与经验回写层；
- 权威事实源、状态、版本、新鲜度和敏感级别；

A1 完成只证明规范和自应用成立，不代表模板已经通过正式业务项目验证。

#### A2：正式业务参考工程验证准备（当前，active）

当前子步骤：**YouYu 高保真确认与工程规格准备**。

已完成：

- YouYu 产品工程目录、项目 Context 和工程治理入口已经建立；
- `YouYu-iOS/` 已确认为唯一正式 iOS 工程，并正式放弃 mPaaS；
- iOS 真机 Debug 构建、安装和启动已通过，服务地址改为构建配置注入；
- 服务端 Maven 构建、App/Gateway 基础接口和内部身份短时签名验证已完成；
- 采集工程依赖修复和构建已完成；
- 既有 GitHub Actions 已通过服务端、采集工程和 iOS 的远程重复检查；
- 首个正式业务切片“手机号验证码登录注册与个人资料管理”已经确认；
- 产品范围、不做清单、业务规则和验收断言已经完成；
- 用户流程、信息架构、页面清单和交互说明已经确认；
- YouYu 已采用统一数据库基础规范，Framework 已登记候选数据库规范；
- 高保真候选已修订至 `v0.1.0-draft.6`，候选检查为 `conditional_pass`；
- 主导航已对齐正式 iOS 工程，空入口、重复事实源、已替代图标和临时预览地址固化问题已关闭；
- 登录页覆盖需要确认协议和已确认当前协议版本、无需重复勾选两种候选状态；
- 候选原型检查已从完整工程构建中拆分为独立轻量工作流；
- 完整工程检查已改为按代码路径触发；
- 候选原型检查覆盖 JavaScript、JSON、版本、精确主导航、协议双状态、唯一事实源、本地引用和页面访问。

当前阻塞：

- 高保真候选尚未由维护者逐页批准；
- 最近候选原型远程运行因 GitHub Actions 账户账单或额度限制而在执行前未启动；维护者已决定暂缓远程成功证据，恢复服务后再重跑；
- 账号、个人资料和验证码表、既有表差异、OpenAPI、错误码、示例数据和客户端映射尚未完成；
- Git 历史敏感信息尚未轮换、清理和建立扫描检查关卡；
- 共享会话存储和下游网络隔离尚未证明；
- 采集工程尚未建立“候选数据 → 人工审核 → 正式钓点数据”的运行控制；
- iOS 测试/生产环境、真机明暗模式、网络异常和服务异常验收尚未完成。

状态边界：

- YouYu 正式业务实现为 `blocked`；
- YouYu 正式业务参考工程验证为 `not_started`；
- 高保真 HTML 演示不证明真实登录、注册或账号安全；
- Context 模板继续为 `candidate`；
- 数据库基础规范在 Framework 中继续为 `candidate`；
- YouYu 项目级轻量检查不等于 Harness 里程碑 B 已经开始；
- Harness 里程碑 B 尚未正式开始；
- 本轮开发进度同步不发布 v0.1.7，稳定版本保持 v0.1.6。

仍需完成：

- 维护者逐页批准高保真候选；
- 建立账号、资料和验证码表及既有表差异清单；
- 建立 OpenAPI、错误码、示例数据和客户端字段映射；
- 关闭或形成可审计隔离结论：安全、共享会话、网络隔离、采集待审和 iOS 真机体验；
- 建立正式项目、阶段和任务 Context；
- 配置修改边界、检查关卡和人工确认节点；
- 完成受控业务实现和静态、运行、模拟用户三层验证；
- 记录 Context 填写成本、人工修正、遗漏、冲突和任务成本；
- 根据完整业务验证修正规范和模板；
- 判断候选资产是否达到 `single_project_validated`；
- 由项目维护者批准里程碑 A 退出。

里程碑 A 退出标准：一次真实业务切片完整使用 Context，形成可追溯的产品、体验、工程、任务、实现、验证和回写闭环，并由维护者批准退出。

- 阶段进入与退出检查关卡；
- 文件和模块修改边界；
- OpenAPI、Schema、依赖和功能行为约定一致性检查；
- 权限、安全、成本和停止条件；
- 三层验证模板；
- 模拟用户验收与发布前检查关卡分离；
- 变更、发布和回滚清单；
- Markdown 链接、Mermaid、版本、状态、许可证和 Context 元数据检查；
- 第一套可运行检查关卡脚本；
- PR 任务状态、Required Checks、人工批准和实际合并权限的一致性控制；
- Xcode 源码注册、资源注册、真机架构和运行路由候选检查关卡。

退出标准：在参考工程中证明检查关卡能够发现真实问题，并且不会因为过度规则阻止正常任务；文档中的人工检查关卡必须能够约束实际合并行为，而不是仅作为说明存在。

### 里程碑 C：Context 与 Harness 联合验证（尚未开始）

建设内容：

- 使用一个小而完整的前后端业务切片；
- 使用项目、阶段和任务 Context Pack；
- 使用 Harness 检查关卡完成受控实现；
- 覆盖前端、后端、OpenAPI、Schema、依赖和联调；
- 记录 Context 缺失、人工修正、任务成功率、成本和验证结果；
- 回写 Framework 缺陷和模板改进。

退出标准：Context 与 Harness 形成一次完整闭环，不只是文档规范；前端、后端、接口约定和数据结构约定能够在同一任务链路中被验证。
