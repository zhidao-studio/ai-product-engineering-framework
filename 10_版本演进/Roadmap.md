# Roadmap

> 路线图遵循“先验证方法，再封装能力，最后自动编排”。发布版本使用语义化版本；`A/B/C` 是 v0.2.0 内部开发里程碑，`A1/A2` 是里程碑 A 的工作段。

版本规则：[版本管理规范](版本管理规范.md)。当前执行事实以 [Framework 项目 Context](../12_框架项目Context/README.md) 为准。

## v0.2.0：Context 与 Harness 可执行化（开发中）

目标：让真实 AI 任务获得正确 Context，在明确边界和检查关卡下完成，并留下可验证、可回写的证据。

### 里程碑 A：Context 可执行化（当前）

#### A1：规范、候选模板与 Framework 自应用

状态：`completed`

已完成：

- 框架、项目、阶段、任务四级 Context；
- 独立决策与经验回写层；
- 权威事实源、状态、版本、新鲜度和敏感级别；
- Context Pack 装配、裁剪、预算和安全规则；
- 冲突、缺失、过期、证据和经验回写；
- 完整性检查清单和候选模板；
- Framework 自应用基线，历史评分 93/100。

A1 只证明规范和自应用成立，不证明候选模板经过真实业务验证。

#### A2：正式业务参考工程验证

状态：`active`

当前子步骤：**YouYu v0.1.1 服务端小版本质量与安全验证**。

已完成：

- 产品范围、业务规则、体验和高保真候选；
- 工程差异、旧代码与 SQL 引用扫描；
- App 用户与管理员账号域隔离决策；
- 数据库基础规范采用；
- 关键参数人工确认；
- 四张 App 账号域表迁移、触发器与回滚；
- v0.1.1 正式 OpenAPI 和错误码；
- 服务端和网关受控实现；
- 真实外部能力失败关闭；
- 维护者服务端先行例外决策；
- TASK-011 验证任务。

当前必须完成：

1. OpenAPI 与代码静态一致性；
2. Maven 测试与打包；
3. MySQL 迁移、约束、触发器和回滚；
4. Gateway、App、共享会话和内部身份联合运行；
5. 验证码错误、过期、使用、频控、并发和协议场景；
6. 日志与敏感信息检查；
7. 修复运行验证发现的问题；
8. 高保真逐页批准；
9. iOS 网络、Keychain、页面状态和真机验收；
10. 模拟用户端到端验收；
11. 失败、成本、人工修正和 Framework 改进回写；
12. 人类责任人批准里程碑结论。

状态边界：

- YouYu 服务端基础实现为 `implemented_not_validated`；
- YouYu 正式业务验证为 `not_started`；
- 高保真仍为候选，iOS 实现仍未开始；
- Context 模板和数据库规范保持 `candidate`；
- Harness 里程碑 B 尚未开始；
- GitHub Actions 外部阻塞未恢复时，不得伪造远程成功证据。

A2 退出条件：

- 真实端到端业务切片完成静态、数据库、运行、安全和模拟用户验收；
- 项目、阶段和任务 Context 能由新 Agent 独立恢复；
- 修改边界和检查关卡在实际任务中生效；
- 失败、人工修正、成本和遗漏有记录；
- Framework 改进完成回写；
- 明确的人类责任人批准。

### 里程碑 B：Harness 可执行化

状态：`not_started`

计划能力：

- 阶段检查关卡；
- 修改范围和依赖白名单；
- OpenAPI、Schema 和代码一致性检查；
- 权限、环境和密钥边界；
- 自动检查与人工确认编排；
- 失败停止、回滚和证据归档。

启动条件：里程碑 A 完成真实参考工程验证，并明确哪些规则值得自动化。

### 里程碑 C：Skills、Agents 与 Loop

状态：`not_started`

计划能力：

- 将已验证方法封装为 Skills；
- 建立角色化 Agents 和受控交接；
- 建立任务、阶段、产品和框架四级反馈闭环；
- 建立失败归因、成本、指标和资产成熟度管理。

启动条件：Context 和 Harness 已在真实任务中稳定运行。

## 当前最高优先级

```text
YouYu TASK-011验证
→ 修复构建与运行问题
→ 高保真人工批准
→ iOS正式实现
→ 真机与模拟用户验收
→ 回写Framework
→ 判断里程碑A退出与候选资产成熟度
```

Framework 仓库采用专有许可，但当前仍为 Public；维护者需要手动切换为 Private 才能形成真正访问控制。
