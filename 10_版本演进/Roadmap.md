## v0.2.0：Context 与 Harness 可执行化

目标：让一次真实 AI 任务获得正确 Context，在明确边界和检查关卡下完成，并留下可验证、可回写的证据。

### 里程碑 A：Context 可执行化

#### A1：规范、候选模板与 Framework 自应用

状态：`completed`

已完成：

- 框架、项目、阶段和任务四级 Context；
- 独立决策与经验回写层；
- 权威事实源、状态、版本、新鲜度和敏感级别；
- Context 装配、裁剪、预算、冲突和安全规则；
- 完整性检查和候选模板；
- Framework 自应用基线。

A1 只证明规范和自应用成立，不代表候选模板已经通过真实业务验证。

#### A2：正式业务参考工程验证

状态：`active`

当前子步骤：**YouYu v0.1.3 运行验证**。

已完成：

- 产品规则、体验定义和高保真候选；
- 工程差异、旧代码与 SQL 引用扫描；
- App 用户与管理员账号域隔离；
- 数据库基础规范、关键参数和服务端先行例外；
- 四张账号域表、历史迁移、v0.1.3 审计触发器修订和安全回滚；
- v0.1.3 OpenAPI、错误码和字段映射；
- 手机号验证码登录注册服务端实现；
- 验证码 `SELECT ... FOR UPDATE` 行锁、失败次数和单次消费修订；
- App/Admin Profile、数据库默认值、SQL 参数日志和 Gateway 地址修订；
- 协议数组、Unicode 长度、Trace ID 和短信适配边界修订；
- YouYu Project Context Pack、TASK-011、TASK-012 和 SERVER-CHECK-003；
- 本地等价检查脚本和调用同一脚本的 GitHub Actions；
- YouYu 静态复核 `passed`。

当前必须取得：

1. Maven 测试与打包实际成功日志；
2. MySQL 基线迁移、v0.1.3 修订、审计行为、约束和回滚成功日志；
3. Gateway、App、MySQL、Redis 和内部身份联合运行；
4. 并发错误验证码 `LOCKED:5` 与并发正确验证码单次消费证据；
5. 协议回滚、登录、资料、退出和日志安全结果；
6. Redis 停机、网络分区和配置不一致验证；
7. 网络来源频控；
8. 高保真逐页批准；
9. iOS 网络、Keychain、页面和真机验收；
10. 模拟用户端到端验收；
11. 失败、成本、人工修正和 Framework 改进回写；
12. 人类责任人批准里程碑结论。

状态边界：

- YouYu 服务端实现：`implemented_not_runtime_validated`；
- 静态复核：`passed`；
- 运行验证：`blocked_external_runtime_evidence`；
- 正式业务验证：`not_started`；
- 高保真：`draft_for_confirmation`；
- iOS 正式实现：`not_started`；
- Context 模板与数据库规范：`candidate`；
- Harness B：`not_started`；
- 检查定义不得写成运行通过。

A2 退出条件：

- 真实业务切片完成静态、数据库、运行、安全和模拟用户验收；
- 项目、阶段和任务 Context 可被新 Agent 独立恢复；
- 修改边界和检查关卡在实际任务中生效；
- 失败、人工修正、成本和遗漏有记录；
- Framework 改进完成回写；
- 人类责任人批准。

### 里程碑 B：Harness 可执行化

### 里程碑 C：Skills、Agents 与 Loop

状态：`not_started`

计划能力：

- 将已验证方法封装为 Skills；
- 建立角色化 Agents 和受控交接；
- 建立任务、阶段、产品和框架四级反馈闭环；
- 建立失败归因、成本、指标和资产成熟度管理。

启动条件：Context 和 Harness 已在真实任务中稳定运行。
