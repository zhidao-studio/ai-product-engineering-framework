## 3. 发现的问题与处理

### 3.1 当前状态重复维护

**问题：** README、AGENTS、Roadmap、项目 Context、阶段 Context和历史报告同时描述进度，读者难以判断哪个文件代表现在。

**处理：**

- `12_框架项目Context/README.md` 成为当前运行状态唯一入口；
- README 和 AGENTS 只保留必要摘要；
- Roadmap 只负责目标、工作段和退出标准；
- CHANGELOG 只负责版本边界和未发布变化；
- 当前阶段 Context 负责阶段产物、任务、风险和退出检查。

### 3.2 历史 `partial_pass` 与当前 `not_started` 容易混淆

**问题：** `REF-CHECK-001` 和早期任务保留 YouYu `partial_pass` 判断，后续 TASK-006 又将正式业务验证修正为 `not_started`。

**处理：**

- 保留历史原始证据，不追溯篡改；
- 在历史文件顶部增加“历史快照”说明；
- 明确历史 `partial_pass` 只表示 PR #1 初步工程证据；
- 当前正式业务参考工程验证继续为 `not_started`；
- 当前状态统一链接项目 Context 和参考工程入口。

### 3.3 阶段 `active` 与当前执行 `blocked` 混写

**问题：** 里程碑 A 仍有效，但正式验证准备因外部依赖无法继续，仅使用一个 `status` 字段会产生误解。

**处理：**

```yaml
status: active
execution_status: blocked
```

同时增加 `current_work_segment`、阻塞原因和解除条件。

### 3.4 Context 缺少仓库基线

**问题：** 项目和阶段 Pack 虽有版本号，但无法精确说明本次复核依据哪个远程提交。

**处理：**

- 项目 Context 增加 `source_commit`；
- 阶段 Context 增加 `baseline_commit`；
- 当前基线固定为 `ecbad719073fd30d6aba0c3ddc7220751ceba942`；
- 版本规范和 Context 规范补充基线字段含义。

### 3.5 历史 93/100 被误认为当前成绩

**问题：** CTX-CHECK-001 的 93/100 是早期自应用基线评分，不能证明 v0.1.5 全仓仍为同一分数。

**处理：**

- 明确 93/100 只属于历史快照；
- 完整性清单新增工作段、执行状态、历史快照和基线提交检查；
- 不在本次发布中制造未经完整自动检查支持的新评分。
