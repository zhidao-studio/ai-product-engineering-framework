# 版本管理规范

> 本规范定义 AI Product Engineering Framework 的稳定发布版本、开发里程碑和 Context Pack 版本，避免不同编号承担相互冲突的含义。

## 1. 稳定发布版本

框架发布版本采用：

```text
v主版本.次版本.修订版本
例如：v0.1.1
```

| 位置 | 含义 | 典型变化 |
|---|---|---|
| 主版本 | 框架进入新的兼容性阶段 | v1.0 形成稳定的跨平台可验证框架 |
| 次版本 | 新增一组达到退出标准并经过真实验证的能力 | v0.2.0 完成 Context 与 Harness 可执行化 |
| 修订版本 | 在既有版本范围内复核、修正和完善 | v0.1.1 修正文档一致性、目录、链接和版本治理 |

在 v1.0 之前，次版本仍可能包含较大的能力演进；任何改变宪法、三平面模型或生命周期的变化仍必须形成设计决策。

## 2. 修订版本边界

以下变化通常增加修订版本：

- 修正文档错误、断链、过期目录和不一致术语；
- 收敛重复事实源，但不改变已批准的核心模型；
- 补充导航、版本说明和检查清单；
- 对现有原则、边界和模型进行更准确的表达；
- 完成全仓 Review 并修复发现的问题；
- 修正示例、状态枚举、元数据字段和模板歧义。

以下变化不应只增加修订版本：

- 增删或重新定义产品生命周期阶段；
- 改变 Context、Harness、Skills、Agents、Loop 的核心边界；
- 转移人和 AI 的最终责任；
- 新增完整的可执行能力族；
- 引入不兼容的模板、门禁或平台契约。

## 3. 建设阶段不是发布版本

`v0.2-A`、`v0.2-B`、`v0.2-C` 是目标版本 v0.2.0 的建设阶段，不是独立发布版本。

推荐写法：

```text
当前稳定版本：v0.1.1
目标发布版本：v0.2.0
当前建设阶段：v0.2-A / Context 可执行化
```

只有目标版本达到 Roadmap 退出标准后，才能发布为稳定版本。

## 4. Context Pack 版本

Context 资产使用独立字段：

```yaml
context_pack_version: 0.2-A.2
stable_release: v0.1.1
target_release: v0.2.0
source_commit: <commit-sha>
```

字段含义：

- `context_pack_version`：当前 Pack 自身结构或内容版本；
- `stable_release`：当前已经确认的框架稳定版本；
- `target_release`：当前开发工作计划进入的发布版本；
- `source_commit`：Pack 对应的 Git 基线。

不得使用含义不清的通用 `version` 字段同时表达上述多个概念。

## 5. 成熟度与版本

版本号不代表仓库中的全部资产已经稳定。每项模板、门禁和 Skill 仍需标明成熟度：

```text
候选 → 单项目已验证 → 跨项目已验证 → 稳定 → 已废弃
```

例如，仓库稳定版本可以是 v0.1.1，但 `main` 分支中同时存在面向 v0.2.0 的候选 Context 模板。README、Roadmap 和 CHANGELOG 必须明确区分二者。

## 6. 版本事实源

版本信息的权威顺序：

1. 根目录 `VERSION`：当前稳定发布版本；
2. `CHANGELOG.md`：已发布版本边界和未发布变化；
3. `10_版本演进/Roadmap.md`：目标版本、建设阶段和退出标准；
4. 本规范：编号和发布规则；
5. README、AGENTS.md 和项目 Context：面向不同受众的状态摘要。

摘要文件不得自行创造与 `VERSION`、CHANGELOG 或 Roadmap 不一致的版本状态。

## 7. CHANGELOG 规则

`CHANGELOG.md` 必须区分：

1. **未发布**：当前建设阶段中尚未形成稳定版本的能力；
2. **已发布版本**：已经完成复核并明确版本边界的内容。

同一项变化不得同时出现在“未发布”和已发布版本中。

## 8. 发布检查

发布修订版本前至少确认：

- [ ] `VERSION` 已更新；
- [ ] README、AGENTS.md、CHANGELOG、Roadmap 和项目 Context 一致；
- [ ] 版本范围没有混入下一次版本的新能力声明；
- [ ] 全仓 Review 报告已提交；
- [ ] 核心内部链接和图片存在；
- [ ] 已知 P0 问题全部关闭；
- [ ] 未完成能力明确标为候选、开发中或待验证；
- [ ] 不改变核心模型，或已有新的设计决策支持变化；
- [ ] 提交信息使用中文并说明发布或修订范围。

## 9. 当前版本基线

```text
v0.1.0：框架立项初版
v0.1.1：全仓 Review 与治理修订，当前稳定版本
v0.2.0：Context 与 Harness 可执行化，开发中
```
