# 版本管理规范

> 本规范定义 AI Product Engineering Framework 的稳定发布版本、开发里程碑、工作段和 Context Pack 版本，避免不同编号承担冲突含义。

## 1. 稳定发布版本

发布版本采用：

```text
v主版本.次版本.修订版本
例如：v0.1.6
```

上述 `v0.1.6` 只是格式示例，不表示当前稳定版本。当前稳定版本必须从根目录 `VERSION` 读取。

| 位置 | 含义 | 典型变化 |
|---|---|---|
| 主版本 | 进入新的兼容性阶段 | v1.0 形成稳定框架 |
| 次版本 | 新增一组达到退出标准且经过真实验证的能力 | v0.2.0 完成 Context 与 Harness 可执行化 |
| 修订版本 | 在既有能力范围内复核、修正、同步和治理 | 事实源、参考工程和发布治理修订 |

任何改变宪法、三平面模型、生命周期、人类最终责任或许可边界的变化都必须建立设计决策。

## 2. 修订版本边界

通常增加修订版本：

- 修正文档、断链、过期状态和术语；
- 收敛重复事实源而不改变核心模型；
- 全仓 Review 与确定性缺陷修复；
- 修正状态枚举、元数据和模板歧义；
- 同步真实参考工程结果，但不提升候选成熟度；
- 补充历史快照、Context 基线和发布检查；
- 加固现有自动检查，而不引入新的完整能力族。

不应只增加修订版本：增删生命周期阶段；改变 Context、Harness、Skills、Agents、Loop 核心边界；转移人和 AI 的最终责任；引入新的完整能力族；改变根本许可模式。

## 3. 里程碑和工作段不是发布版本

- `A/B/C` 是目标版本内部建设里程碑；
- `A1/A2` 是里程碑内部工作段；
- 它们不进入根目录 `VERSION`；
- 只有 Roadmap 退出条件完成并经人类责任人批准，目标版本才能发布为稳定版本。

动态状态只保存在项目 Context 和 README 摘要中：稳定版本从 VERSION 读取，目标发布版本、当前里程碑和工作段从项目 Context 读取。

## 4. Context Pack 版本

```yaml
project_context_pack_version: <pack-version>
stable_release: <VERSION对应标签>
target_release: <目标版本>
source_commit: <本仓库最近复核提交>
baseline_commit: <本仓库任务或阶段开始基线>
reference_repository: <参考仓库>
reference_source_commit: <参考仓库提交>
```

必须区分 Context Pack 版本、稳定发布版本、目标版本、本仓库来源提交、本仓库基线提交和参考仓库提交。不得将参考仓库 SHA 写入本仓库 `source_commit`。Pack 元数据提交可以晚于其 `source_commit`，不代表业务基线变化。

## 5. 成熟度与版本

版本号不代表仓库中的全部资产已经稳定。每项模板、检查关卡和 Skill 继续使用：

```text
candidate
→ single_project_validated
→ cross_project_validated
→ stable
→ deprecated
```

稳定版本发布不能自动提升任何候选资产成熟度。

## 6. 版本与状态事实源

版本权威顺序：

1. 根目录 `VERSION`；
2. `CHANGELOG.md`；
3. `10_版本演进/Roadmap.md`；
4. 本规范；
5. README 摘要。

当前运行状态权威顺序：

1. `12_框架项目Context/README.md`；
2. 当前阶段 Context；
3. 当前任务 Context Pack；
4. README 与 Roadmap 摘要；
5. 历史任务、验证和发布报告。

`AGENTS.md` 只保存长期规则，不保存当前稳定版本、当前工作段、当前参考工程版本、阻塞或某次检查结果。

## 7. CHANGELOG 规则

`CHANGELOG.md` 必须区分未发布目标版本和已发布修订版本。同一能力不得在未发布和稳定版本中被重复表述为两套已完成稳定能力。修订版本可以记录开发能力当前状态，但不能把未完成能力写成稳定能力。

## 8. 历史快照规则

历史任务、验证和发布报告：保留发生时的版本、术语、判断和证据；不追溯篡改原始结论；顶部标明历史快照；链接当前项目 Context 或后续修正任务；标明被哪个后续版本、任务或验证替代。

## 9. 发布检查

发布修订版本前至少确认：

- [ ] `VERSION` 已更新；
- [ ] README、CHANGELOG、Roadmap、项目 Context 一致；
- [ ] AGENTS 未维护动态当前版本和工作段；
- [ ] 目标版本和里程碑没有被误写为稳定发布；
- [ ] 参考工程版本、实现状态和正式验证状态一致；
- [ ] Context 模板和数据库规范成熟度一致；
- [ ] Harness 后续里程碑状态一致；
- [ ] Framework `baseline_commit`、`source_commit` 可由 `git cat-file -e` 解析；
- [ ] 参考仓库提交与本仓库提交使用不同字段；
- [ ] 提供参考仓库时校验参考 SHA 的仓库归属；
- [ ] 未完成能力标为候选、阻塞或待验证；
- [ ] P0 已关闭，或明确不属于本次范围且未宣称通过；
- [ ] 核心模型未改变，或已有设计决策；
- [ ] 历史文件与当前状态入口边界明确；
- [ ] `bash scripts/check-release-state.sh` 可执行；
- [ ] 提交信息使用中文并准确说明范围。
